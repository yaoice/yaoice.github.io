总体对比

JKE OpenShift
集群管理 中（提供基础集群全生命周期管理、容灾备份功能），跨多个云纳管Kubernetes集群；创建集群、添加节点是基于ssh协议，比ansible轻量；提供集群巡检服务，并自动生成巡检报告； 强（提供基础集群全生命周期管理功能、高级集群管理、集群防护、容灾备份功能），跨多个私有云和公共云集中创建、更新和删除Kubernetes集群；在创建或更新集群时，利用红帽Ansible®自动化平台集成自动执行各种任务，例如配置云定义存储和静态 IP 地址、更新网络组件（如防火墙或负载平衡器），以及完成其他任务；利用红帽Ansible自动化平台自动修复策略违规，并收集集群审核信息以进行分析；通过 Submariner 提供交叉集群网络基础架构，从而实现直接加密通信
流水线 强（基于Jenkins实现的k8s原生CI/CD方案，多形式应用发布流程(基于环境隔离、基于环境限制特定角色)、多形式缓存加速应用构建） 强（基于Tekton的k8s原生CI/CD方案，使用标准的Tekton CRD定义作为容器运行的管道，并按需扩展，有很多针对提示性的模版；支持GitOps工作流；支持knative的无服务器应用）
监控 强（基于prometheus、victorymetric方案，支持集群、应用监控、外部监控实例管理、告警功能、vGPU监控） 中（基于prometheus、thanos、victorymetric方案， 查看和管理指标、警报，并查看监控仪表板;自定义监控、告警；nvidia GPU仪表板）
日志 强（基于EFK方案，集成度高，支持日志检索、下载、多维度查询、日志告警功能） 中（基于EFK方案，）
镜像仓库 中（提供开源Harbor集成，支持外部镜像仓库集成） 强（红帽®Quay 是一个安全至上并可扩展的私有镜像仓库平台，用于管理分布于全球各地的数据中心和云环境中的内容。它提供了一个单一且有弹性的内容存储库，用于跨红帽 OpenShift®和Kubernetes集群将容器化软件交付到开发和生产环境。红帽Quay是一个面向企业的分布式高可用性容器镜像仓库）
应用商店 弱（基于Helm chart实现，提供应用模版、应用审核、应用管理功能） 强（支持helm chart，还有Operator应用商店）
安全 弱（调研NeuVectord云原生安全平台） 强，红帽Kubernetes高级集群安全防护（ACS）是领先的Kubernetes原生安全平台；漏洞管理(识别修复漏洞)、网络分段(网络流量、网络策略)、配置管理(最佳实践强化应用安全)、合规审计(根据CIS基准、NIST、PCI、HIPAA)、风险预测(根据k8s声明性资源的上下文)、检测和响应(使用规则、允许列表和基线来识别可疑活动，并采取行动阻止攻击)
WebIDE 中（基于code-server方案，提供webide实例、模版功能，支持VS Code开发界面） 强（支持vs code和IDEA Intellij作为开发界面）
机器学习 中（提供GPU调度、vCuda显存隔离能力） 强（OpenShift®数据科学是一个面向混合云的开源机器学习平台；集成了开源工具和商业合作伙伴的应用，让组织可以快速地构建和部署AI/ML模型；OpenShift数据科学会跟踪对Jupyter、TensorFlow、PyTorch和其他开源AI技术进行的更改）
容器存储 弱（对接外部存储） 强（支持所有基本存储类、S3兼容对象存储、数据复制和灾难恢复功能等重要功能）
成本 中 高
资源占用 低 高（自身平台组件太多）

OpenShift

OpenShift简介

红帽OpenShift是业内领先的混合云应用平台，它基于 Kubernetes，整合了多项经过测试且深受信赖的服务，可减少开发、现代化、部署、运行和管理应用等过程中产生的摩擦。 OpenShift 可以跨公共云、本地、混合云或边缘架构提供一致体验。OpenShift有开源版和企业版之分。

OpenShift版本分类

OpenShift开源社区版本：也叫OKD(OpenShift Kubernetes Distribution)，基于k8s的发行版. 部署出来的平台左上角有OKD标识.

OpenShift企业版 - OpenShift云服务版(OpenShift Cloud Services)：OpenShift可作为主流云提供商的一款统包式应用平台来提供。在您选择的云提供商平台上，大规模构建、现代化和部署应用。 部署出来的平台左上角有OpenShift Dedicated标识.

OpenShift企业版 - OpenShift自助管理版(OpenShift Platform Plus)：OpenShift平台Plus可以安装在本地、云、托管云或边缘等环境，在混合型基础架构中提供一致的用户体验、管理和安全防护。 红帽OpenShift平台Plus，其包含了红帽OpenShift容器平台(OpenShift Container Platform)的所有功能，以及高级集群管理和安全防护、数据管理基础，以及一个企业容器镜像仓库。

通过部署出来的openshift环境左上角的logo也可以区分出来：

  • okd：openshift开源版
  • openshift dedicated：openshift公有云托管版本
  • openshift：openshift私有化版本

OpenShift架构

OpenShift架构由这些组件组成：

  • master节点：运行API、认证鉴权、元数据存储、调度、健康检查、动态扩缩容等服务
  • node节点：运行应用的节点
  • 持久化存储
  • 私有镜像仓库
  • 路由层(routing layer)
  • 服务层(service layer)
  • IaaS基础设施：物理机、虚拟机、私有云、公有云、混合云

OpenShift环境

  • OpenShift版本：4.12.1
  • Kubernetes版本：1.25.4

OpenShift核心功能

OpenShift Clusters

  • OpenShift Cluster：提供基础集群全生命周期管理功能、高级集群管理、集群防护、容灾备份功能
  • JKE Cluster：提供基础集群全生命周期管理、容灾备份功能

OpenShift Pipelines

OpenShift Pipeline功能完全基于Tekton实现，Tekton是开源的CI/CD框架. 具备有针对提示性的模版供选择、自定义流水线优化模块

OpenShift构建：基于源码的构建(S2i)，或Dockerfile的构建；提供针对提示性的模版选择

对比JKE： JKE流水线基于Jenkins实现，利用k8s pod的动态能力实现Jenkins slave自动化添加，运行任务

JKE构建/部署提供轻量化工作流功能，基于谷歌开源的skaffold，集成了环境/角色隔离功能

OpenShift优点：自定义yaml/json填写，非常灵活；有很多针对提示性的模版 JKE优点：多形式应用发布流程(基于环境隔离、基于环境限制特定角色)、多形式缓存(应用加速构建场景，项目缓存/集群缓存)

OpenShift Monitor

OpenShift Monitor基于prometheus、thanos、victorymetric开源方案实现

对比JKE monitor，基于prometheus、victorymetric开源方案实现

JKE monitor总体功能超过OpenShift，OpenShift缺少应用监控、外部监控实例管理功能

OpenShift logging

OpenShift logging基于开源EFK方案实现

对比JKE logging，基于EFK方案

JKE logging总体功能超过OpenShift，OpenShift集成度较低、缺少日志告警功能

OpenShift Image Registry

  • OpenShift Registry：企业级镜像仓库
  • JKE registry：提供开源Harbor集成，支持外部镜像仓库集成

Azure

Azure DevOps

Azure DevOps简介

微软开发的服务平台，它提供了多种工具，可用于更好地进行团队协作。它还具有用于自动构建过程，测试，版本控制和程序包管理的工具

Azure DevOps 提供了6个主要模块：

  • Azure Boards：需求迭代模块，利用经过验证的敏捷工具，计划、跟踪和讨论团队工作，以更快地为用户提供价值
  • Azure Pipelines：流水线模块，使用适用于任何语言、平台和云的CI/CD构建、测试和部署。连接到GitHub或任何其他Git提供者并实现持续化部署
  • Azure Repos：代码仓库模块，获得无限的云托管私有Git存储库，并通过拉取请求和高级文件管理进行协作，以构建更好的代码
  • Azure Test Plans：测试计划模块，使用手动和探索性测试工具进行测试和交付
  • Azure Artifacts：制品模块，创建、托管和共享团队的软件包，方便与CI/CD流水线结合
  • Github Advanced Security for Azure DevOps：是一种本地于开发人员工作流程的应用程序安全测试服务。它赋予开发人员、安全团队和运营（DevSecOps）团队优先考虑创新并提高开发人员生产力的能力， 而不会牺牲安全性。该服务提供了一系列安全检查和漏洞扫描工具，使开发者可以在代码编写的早期阶段发现和修复潜在的安全问题，从而降低安全风险，保护应用程序免受攻击和数据泄露。

除此之外，扩展市场上还有超过1000个应用和工具可供选择.

Azure DevOps核心功能

Azure Boards

跟其它敏捷管理的竞品功能上大同小异，工作项、Sprints、发布计划、看板等版块都是标配功能，使用起来也比较简洁

Azure Repos

具备代码仓库基础功能，涉及文件展示、commmit展示、Pushes过滤、分支管理、tag管理、pull requests管理

Azure Pipelines

  • 流水线：pipeline as code风格，由众多task组成，图形化编排能力较弱，偏持续集成
  • 环境：可以针对特定环境(映射对应k8s命名空间)的资源进行管理，并设定审批、分支控制、操作时间，适用于所有pipelines
  • 发布：更像是增强型的流水线，图形化编排能力强，可与环境串联起来，偏持续部署
  • 共享库：多个流水线共享的变量组
  • 任务组：通用的构建/部署步骤
  • 部署组：并行部署的目标机器

总体来说，cicd能力比较强，要说缺点的话，技术栈闭源勉强算一个

Azure Test Plans

Azure Artifacts

制品功能不是非常丰富，比如缺少rpm、deb包管理等能力
