зҺҜеўғ

  • OS: CentOS 7.5
  • Kubernetes v1.11.6
  • Etcd 3.3.10
  • Docker 1.13.1

д»Җд№ҲжҳҜAdmissionWebhook

д»Җд№ҲжҳҜAdmissionWebhookпјҢе°ұиҰҒе…ҲдәҶи§Јk8sдёӯзҡ„admission controller, жҢүз…§е®ҳж–№зҡ„и§ЈйҮҠжҳҜпјҡ admission controllerжҳҜжӢҰжҲӘ(з»ҸиҝҮиә«д»ҪйӘҢиҜҒ)API ServerиҜ·жұӮзҡ„зҪ‘е…іпјҢе№¶дё”еҸҜд»Ҙдҝ®ж”№иҜ·жұӮеҜ№иұЎжҲ–жӢ’з»қиҜ·жұӮгҖӮ

з®ҖиҖҢиЁҖд№ӢпјҢе®ғеҸҜд»Ҙи®ӨдёәжҳҜжӢҰжҲӘеҷЁпјҢзұ»дјјwebжЎҶжһ¶дёӯзҡ„middlewareгҖӮ

k8sй»ҳи®ӨжҸҗдҫӣеҫҲеӨҡеҶ…зҪ®зҡ„admission controllerпјҢйҖҡиҝҮkube-apiserverеҗҜеҠЁе‘Ҫд»ӨеҸӮж•°еҸҜд»Ҙ жҹҘзңӢеҲ°ж”ҜжҢҒзҡ„admission controller pluginжңүе“ӘдәӣгҖӮ

[root@node220]# kube-apiserver --help |grep enable-admission-plugins

# ж”ҜжҢҒзҡ„pluginжңүеҰӮдёӢ
AlwaysAdmit, AlwaysDeny, AlwaysPullImages, 
DefaultStorageClass, DefaultTolerationSeconds, DenyEscalatingExec, 
DenyExecOnPrivileged, EventRateLimit, ExtendedResourceToleration, 
ImagePolicyWebhook, Initializers, LimitPodHardAntiAffinityTopology, 
LimitRanger, MutatingAdmissionWebhook, NamespaceAutoProvision, 
NamespaceExists, NamespaceLifecycle, NodeRestriction, 
OwnerReferencesPermissionEnforcement, PersistentVolumeClaimResize, 
PersistentVolumeLabel, PodNodeSelector, PodPreset, PodSecurityPolicy, 
PodTolerationRestriction, Priority, ResourceQuota, SecurityContextDeny, 
ServiceAccount, StorageObjectInUseProtection, ValidatingAdmissionWebhook.

иҝҷйҮҢenableзҡ„admission-pluginsеҰӮдёӢ

--enable-admission-plugins=PersistentVolumeClaimResize,NamespaceLifecycle,LimitRanger,ServiceAccount,DefaultStorageClass,ResourceQuota,NodeRestriction,ValidatingAdmissionWebhook,MutatingAdmissionWebhook

иҝҷйҮҢдёҚеҜ№жҜҸдёӘpluginиҜҰз»ҶиҜҙжҳҺпјҢзҪ‘дёҠйғҪеҸҜд»ҘжҗңеҲ°зӣёе…іиө„ж–ҷгҖӮ жҖ»дҪ“жқҘиҜҙпјҢadmission-pluginsеҲҶдёәдёүеӨ§зұ»пјҡ

  1. дҝ®ж”№зұ»еһӢ(mutating)
  2. йӘҢиҜҒзұ»еһӢ(validating)
  3. ж—ўжҳҜдҝ®ж”№еҸҲжҳҜйӘҢиҜҒзұ»еһӢ(mutating&validating)

иҝҷдәӣadmission pluginжһ„жҲҗдёҖдёӘйЎәеәҸй“ҫпјҢе…ҲеҗҺйЎәеәҸеҶіе®ҡи°Ғе…Ҳи°ғз”ЁпјҢдҪҶдёҚдјҡеҪұе“ҚдҪҝз”ЁгҖӮиҝҷйҮҢе…іеҝғзҡ„pluginжңүдёӨдёӘпјҡ

дёҖ. MutatingAdmissionWebhook, ValidatingAdmissionWebhook

  • MutatingAdmissionWebhook: еҒҡдҝ®ж”№ж“ҚдҪңзҡ„AdmissionWebhook
  • ValidatingAdmissionWebhook: еҒҡйӘҢиҜҒж“ҚдҪңзҡ„AdmissionWebhook

еј•з”Ёkubernetesе®ҳж–№еҚҡе®ўзҡ„дёҖеј еӣҫжқҘиҜҙжҳҺMutatingAdmissionWebhookе’ҢValidatingAdmissionWebhookжүҖеӨ„зҡ„дҪҚзҪ®:

и§ЈйҮҠдёӢиҝҷдёӘиҝҮзЁӢпјҡ

  1. apiиҜ·жұӮеҲ°иҫҫK8s API Server
  2. иҜ·жұӮиҰҒе…Ҳз»ҸиҝҮи®ӨиҜҒ
    • kubectlи°ғз”ЁйңҖиҰҒkubeconfig
    • зӣҙжҺҘи°ғз”ЁK8s apiйңҖиҰҒиҜҒд№Ұ+bearToken
    • client-goи°ғз”Ёд№ҹйңҖиҰҒkubeconfig
  3. жү§иЎҢдёҖиҝһдёІзҡ„admission controllerпјҢеҢ…жӢ¬MutatingAdmissionWebhookе’ҢValidatingAdmissionWebhook, е…ҲдёІиЎҢжү§иЎҢMutatingAdmissionзҡ„Webhook list
  4. еҜ№иҜ·жұӮеҜ№иұЎзҡ„schemaиҝӣиЎҢж ЎйӘҢ
  5. е№¶иЎҢжү§иЎҢValidatingAdmissionзҡ„Webhook list
  6. жңҖеҗҺеҶҷе…Ҙetcd

дәҢ. Initializers vs AdmissionWebhookеҢәеҲ«

дәҢиҖ…йғҪиғҪе®һзҺ°еҠЁжҖҒеҸҜжү©еұ•иҪҪе…Ҙadmission controllerпјҢ InitializersжҳҜдёІиЎҢжү§иЎҢпјҢеңЁй«ҳе№¶еҸ‘еңәжҷҜе®№жҳ“еҜјиҮҙеҜ№иұЎеҒңз•ҷеңЁuninitializedзҠ¶жҖҒпјҢеҪұе“Қз»§з»ӯи°ғеәҰгҖӮ Alpha Initializersзү№жҖ§еңЁk8s 1.14зүҲжң¬иў«з§»йҷӨдәҶпјҢиҜҰжғ…и§Ғhttps://github.com/kubernetes/apimachinery/issues/60гҖӮ зӣёжҜ”InitializersпјҢе®ҳж–№жӣҙжҺЁиҚҗAdmissionWebhookпјӣMutatingAdmissionWebhookжҳҜдёІиЎҢжү§иЎҢпјҢValidatingAdmissionWebhookжҳҜе№¶иЎҢжү§иЎҢпјҢжҖ§иғҪжӣҙеҘҪгҖӮ

AdmissionWebhookеә”з”ЁеңәжҷҜ

IstioзӣёдҝЎеӨ§е®¶йғҪжңүеҗ¬иҝҮпјҢistioе°ұжҳҜйҮҮз”ЁAdmissionWebhookе®һзҺ°sidecarе®№еҷЁиҮӘеҠЁжіЁе…ҘгҖӮжҲ‘зӣ®еүҚз”ЁеҲ°зҡ„еә”з”ЁеңәжҷҜжңүдёӨдёӘпјҢеҪ“з„¶иӮҜе®ҡиҝҳжңүе…¶е®ғеә”з”ЁеңәжҷҜгҖӮ

  • иҮӘеҠЁжү“ж Үзӯҫ

    жҜ”еҰӮеҗҜеҠЁдёҖдёӘеә”з”ЁпјҢеә”з”ЁеҢ…жӢ¬deploymentгҖҒserviceгҖҒingressпјӣ жҖҺд№Ҳеҝ«йҖҹиҝҮж»ӨеҮәе“Әдәӣиө„жәҗеұһдәҺеә”з”Ё? еңЁk8sдёӯпјҢpodгҖҒserviceгҖҒingress йғҪжҳҜзӢ¬з«Ӣзҡ„иө„жәҗпјҢйҖҡиҝҮз»ҷиҝҷдәӣиө„жәҗжү“дёҠlabelпјҢжҳҜжңҖеҝ«йҖҹзҡ„ж–№ејҸгҖӮ

  • иҮӘеҠЁжіЁе…Ҙsidecarе®№еҷЁ

    еә”з”ЁеҗҜеҠЁеҗҺпјҢеә”з”Ёзҡ„зӣ‘жҺ§гҖҒж—Ҙеҝ—еҰӮдҪ•еӨ„зҗҶпјҹеҖҹеҠ©sidecarе®№еҷЁжіЁе…ҘеҲ°е…¶podдёӯ

ж”¶йӣҶеә”з”Ёж—Ҙеҝ—зҡ„sidecarе®№еҷЁеҸҜд»ҘеғҸдёӢеӣҫжүҖзӨәпјҢеә”з”Ёзӣ‘жҺ§зҡ„sidecarе®№еҷЁзұ»дјј

еҸ‘ж•ЈжҖқз»ҙ

AdmissionWebhookеӨҡйӣҶзҫӨеә”з”Ё

еҰӮжһңжңүеӨҡдёӘk8sйӣҶзҫӨпјҢйӮЈжҳҜдёҚжҳҜиҰҒеңЁжҜҸдёӘйӣҶзҫӨйғҪеҗҜеҠЁдёҖдёӘWebhook APIе‘ўпјҹжүҖд»Ҙе°ұжңүдәҶдёӢйқўзҡ„жһ¶жһ„пјҢжүҖжңүйӣҶзҫӨе…ұдә« дёҖдёӘWebhook APIгҖӮ

cluster c1гҖҒc2дёӯзҡ„Webhookй…ҚзҪ®дјҡжҢҮеҗ‘еҗ„иҮӘйӣҶзҫӨеҶ…йғЁзҡ„serviceпјҢиҝҷдёӘserviceе…¶е®һжҳҜheadless serviceпјҢ е®ғжҢҮеҗ‘зҡ„жҳҜcluster Aзҡ„service(йңҖиҰҒжҡҙйңІз»ҷе…¶е®ғйӣҶзҫӨиғҪеӨҹи®ҝй—®пјҢnodePortд№ҹеҸҜд»Ҙ)пјҢиҝҷж ·жүҖжңүйӣҶзҫӨе°ұе…ұдә«дёҖдёӘWebhook APIдәҶгҖӮ

жҖ»з»“

AdmissionWebhookеҸҜд»ҘеғҸжӢҰжҲӘеҷЁдёҖж ·жӢҰжҲӘK8s apiиҜ·жұӮпјҢиҰҒе®һзҺ°дҝ®ж”№еҠҹиғҪз”ЁMutatingAdmissionWebhookпјҢ е®һзҺ°йӘҢиҜҒеҠҹиғҪз”ЁValidatingAdmissionWebhookгҖӮ

еҶҷдёҖдёӘAdmissionWebhook APIзҡ„иҰҒзӮ№пјҡ

  1. AdmissionReviewз»“жһ„дҪ“ requestиҜ·жұӮдҝЎжҒҜйҖҡиҝҮAdmissionReviewзҡ„Requestеӯ—ж®өеҸҜд»ҘиҺ·еҸ–еҲ°пјӣ responseйҖҡиҝҮAdmissionReviewзҡ„Responseеӯ—ж®өи®ҫзҪ®иҝ”еӣһ

    // AdmissionReview describes an admission review request/response.
type AdmissionReview struct {
    metav1.TypeMeta `json:",inline"`
    // Request describes the attributes for the admission request.
    // +optional
    Request *AdmissionRequest `json:"request,omitempty" protobuf:"bytes,1,opt,name=request"`
    // Response describes the attributes for the admission response.
    // +optional
    Response *AdmissionResponse `json:"response,omitempty" protobuf:"bytes,2,opt,name=response"`
}

  2. mutatingжҳҜйҖҡиҝҮjson patchж–№ејҸе®һзҺ°зҡ„

    еҜ№еә”зҡ„з»“жһ„дҪ“е®ҡд№үеҰӮдёӢпјҡ

    type patchOperation struct {
  Op    string      `json:"op"`
  Path  string      `json:"path"`
  Value interface{} `json:"value,omitempty"`
}

patches = append(patches, patchOperation{
  Op:    "add",
  Path: "/metadata/annotations",
  Value: true,
})
